1) CMS Controls Needed (High Priority)

1.1 Image ALT Management

  • Add alt fields for all CMS-managed images (sliders, products, banners, logos, blog images, etc.).
  • API responses should include localized alt text where applicable (e.g. alt_ar, alt_en).
  • If alt is empty, backend should return a safe fallback value.

1.2 Link rel Attributes from CMS

  • Support per-link flags for nofollow.
  • Recommended: also support noopener, noreferrer, sponsored, ugc.
  • Return structured link metadata so frontend can render proper rel attributes.

1.3 Rich Text Support for Footer/Content Blocks

  • Allow sanitized rich text (or structured content blocks) in CMS fields used in footer/content pages.
  • Must support: hyperlinks, strong/emphasis formatting, lists, and line breaks.

1.4 Product Slug Management

  • Allow editing and locking unique product slugs from CMS.
  • Enforce uniqueness constraints and avoid collisions.
  • When slug changes, provide redirect mapping (301 old slug to new slug).
